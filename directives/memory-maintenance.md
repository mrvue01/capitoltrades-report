# Memory Maintenance

## Purpose

Keep the OpenClaw memory system useful, small, and durable.

## Trigger

Use during heartbeats or periodic cleanup.

## Steps

1. Read recent `memory/YYYY-MM-DD.md` files
2. Identify durable facts, decisions, and preferences
3. Promote only high-signal items into [[MEMORY]]
4. Move project-specific details into dedicated project notes when needed
5. Remove stale or duplicated material from [[MEMORY]]
6. Add links between related notes so the Obsidian graph stays meaningful

## Edge cases

- In shared contexts, avoid copying private information into broadly shared notes
- If a topic is large, create a dedicated note instead of bloating `MEMORY.md`
- If a fact is temporary, keep it in a daily note instead of promoting it

## Outputs

- Updated [[MEMORY]]
- Better-linked project or concept notes
- A cleaner long-term memory graph
